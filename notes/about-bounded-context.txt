board 的所有變化都會同步現在有的 term
model 也是，我們會在專案中，一個 subdomain 中建立新的 models

概念釐清
domain 是整體, project = domain, domain = system
domain 有多個 subdomain 和一個 core domain
subdomain 自身有 bounded-context
subdomain 中可能有多個 bounded-context

概念對應
domain / system
subdomain / services
bounded-context / packages

domains > core-domain/subdomain > bounded-context > model
systems > sub-systems > services (containers in c4) > components

sub-system/subdomain (type: 整合端（web, api, ...）、核心、支援、基底)
bounded-contexts/units (type: 核心業務、對外接口（api, sdk）、應用程式、操作介面 (web, mobile, desktop, ...))

一個領域中有多個子領域，每個子領域會有多個上下文
一個系統中有多個模組，每個模組都會有多個套件，每個套件中有多個元件

在討論一個系統時，會限縮在一個模組中，只要在模組中，上下文之間可能會共享部分概念，比如後端應用層，後端伺服器，前端應用層，前端網頁

後端應用層將探討所有的抽象，並建立假的實作來檢驗
後端伺服器將整合後端應用層，使用真的實作來搭建
前端應用層將探討所有的抽象，並搭建與後端使用的模型，並建立假實作來檢驗
前端網頁將整合前端應用層，並使用真的實作來搭建

搭建系統時，初期只引導用戶在核心服務中建立專案，接著才可以新增支援服務、基底服務、整合服務等
